# Kakeibo「抹茶慢生活」UI 优化设计

## 目标

将 Kakeibo 从当前偏黑白、工具化的界面升级为可爱、柔和、让人愿意每天打开的移动端记账体验。新版采用「抹茶慢生活」视觉方向：暖米白、鼠尾草绿和燕麦色，保持克制、自然、长期耐看的气质。

本次升级覆盖账单、统计、分类、设置、记账弹窗、月份选择、预算、CSV 导入及 WebDAV 配置等全部现有界面。改造属于中度布局升级：允许重组信息层级、卡片、留白和反馈，但保留四页导航、核心操作位置、业务流程、数据结构和功能行为。

## 成功标准

- 四个页面及所有弹层使用一致的颜色、圆角、图标、间距、阴影和交互反馈。
- 产品 UI 不再使用 Emoji，改为统一的圆润线性 SVG 图标。
- 浅色和深色模式均为完整的抹茶主题，并保持清晰的文字与金额对比度。
- 记账、删除、月份切换、预算、CSV 导入导出及 WebDAV 行为保持不变。
- 390px 和 430px 移动端宽度下无横向溢出，长文本和大金额仍可读。
- TypeScript 构建和现有测试通过，新增图标映射与关键交互测试。

## 设计原则

1. **柔和但不含糊**：降低颜色饱和度，同时保留明确的信息层级和状态区分。
2. **可爱但不幼稚**：通过圆润形状、留白和轻动效营造亲和力，不增加无关装饰或游戏化功能。
3. **金额优先**：金额、预算状态和主要操作保持最高可读性，视觉装饰不得干扰财务信息。
4. **一致胜过新奇**：所有页面共享语义变量、图标系统和基础组件，不按页面发明独立样式。
5. **保护现有行为**：UI 层消费现有 hooks、状态与回调，不改数据库、导入规则或业务模型。

## 全局视觉系统

### 颜色

浅色主题使用暖米白页面背景、浅燕麦卡片和鼠尾草绿主色。收入使用柔和青绿，支出和危险操作使用低饱和珊瑚红；预算警告使用柔和琥珀色。

深色主题使用深橄榄灰背景和略浅的苔藓色卡片，避免纯黑。深色模式沿用相同语义色，通过调整明度与边框确保对比度。组件只使用语义 CSS 变量，不直接散落十六进制颜色。

建议的语义变量包括：

- 页面、次级区域、卡片、浮层背景
- 主文字、次级文字、弱提示文字
- 主色及其弱背景、悬停和按压状态
- 收入、支出、警告、成功及对应弱背景
- 边框、分隔线、阴影和遮罩
- 导航激活、导航未激活、输入框和标签状态

### 形状、间距与阴影

- 页面主卡片使用约 22px 圆角。
- 普通卡片和分区使用约 16px 圆角。
- 按钮、输入框和分类图标容器使用约 14px 圆角。
- 标签和小状态使用胶囊形。
- 浅色模式使用低透明度、宽扩散的柔和阴影。
- 深色模式主要依靠细边框和微弱高光分层，避免灰黑阴影造成脏感。
- 页面统一使用移动端安全区和 16px 左右的内容边距，点击区域不小于 44px。

### 字体与数字

继续使用系统中文字体栈，不增加网络字体依赖。金额采用更紧凑、醒目的数字层级；正文减少过度粗体，通过字号、颜色和间距建立层级。大金额必须允许缩放或合理换行，不能挤压相邻操作。

### 图标系统

新增集中式 `Icon` 组件，使用统一的圆润线性 SVG：圆端点、圆连接、统一描边宽度和 `currentColor` 着色。图标系统覆盖：

- 四个底部导航入口和中央记账按钮
- 系统分类与自定义分类
- 商户、日期、月份、导入导出、同步、备份、删除等操作
- 成功、警告、失败和空状态

分类图标放在低饱和淡色圆角容器中。分类数据保存稳定的图标标识，展示层不再依赖 Emoji。无法匹配的旧分类使用统一的通用分类图标，不显示 Emoji 回退。

### 动效与无障碍

保留现有弹簧抽屉和滑动删除，增加轻量按钮按压、页面内容淡入和进度条过渡。动效只用于表达层级与状态，不阻塞操作；系统启用 `prefers-reduced-motion` 时关闭非必要位移与过渡。

文字、金额、状态和焦点需有足够对比度。状态不能只依赖颜色表达；图标配合文本或可访问标签。键盘焦点清晰，图标按钮提供可读的 `aria-label`。

## 页面设计

### 账单页

顶部改为月度总览卡，包含轻量问候语、月份选择、本月支出，以及收入和结余两个次级小卡。CSV 导入保留在头部，但降为次级图标操作。

交易列表按日期分组，使用清晰的日期标题和轻量时间线感。每笔交易显示分类图标、商户或备注、分类和来源信息、金额；支出和收入使用语义色。滑动删除行为不变，删除区域改用统一危险操作样式。

空账单状态使用线性图标、简短中文提示和明确的“记一笔”入口。

### 底部导航

底部导航改为悬浮圆角栏，考虑设备安全区。四个圆润线性图标位于中央操作两侧，激活项使用主色和弱背景。中央记账按钮使用鼠尾草绿，保持全应用最高的操作优先级。

导航结构仍为账单、统计、分类和设置，不改变页面含义。

### 记账弹窗

底部弹窗增加顶部拖动指示、标题和关闭按钮。字段顺序为收支切换、金额、分类、备注、日期、保存。收支切换使用柔和分段控件；金额保持视觉中心；分类使用统一图标网格；保存按钮为明确的主操作。

金额校验改为表单内错误提示。保存成功继续关闭弹窗并刷新现有数据，不改变数据提交逻辑。

### 统计页

保留月份切换、收支结余、预算、六个月趋势、分类支出和商户支出结构，统一放入分区卡片。图表使用抹茶绿表示当前或正常状态、燕麦灰表示历史、珊瑚红表示超预算、琥珀色表示预算参考线。

图表标签、大金额和图例在 390px 宽度下不能重叠。无数据时显示统一空状态，不保留 Emoji。

### 分类页

支出与收入分类拆成两个清晰分区。分类项使用圆角线性图标容器、名称和必要的操作；系统分类和自定义分类通过文字或操作能力区分，不用突兀标签。

新建分类保留低强调虚线按钮。分类表单使用同一底部弹窗模式，图标选择器只展示统一图标库。

### 设置页

设置按“同步与备份”“数据导入导出”等任务分组。WebDAV 状态、测试连接和保存操作使用统一表单与状态反馈。危险操作独立放在柔和红色区域，并保留二次确认。

版本信息保持低强调展示。

### 其他弹层与反馈

月份选择、预算设置、CSV 预览和分类编辑共享统一的底部弹层结构、遮罩、标题区和按钮层级。成功、警告和失败使用页面内提示条或 Toast；确认操作使用一致的确认弹层。原生 `alert` 和 `confirm` 仅在尚未迁移的兜底路径保留，最终可见核心流程不依赖其样式。

## 技术结构

采用设计系统渐进升级方案：

1. 扩充 `theme.css`，建立浅色、深色和语义状态变量。
2. 增加统一的 `Icon` 组件与稳定的图标名称映射。
3. 增加少量可复用 UI 基础单元，例如卡片、图标按钮、提示条、底部弹层外壳和空状态。
4. 按账单与导航、记账流程、统计、分类、设置及辅助弹层的顺序替换现有内联样式。
5. 页面业务组件继续使用现有 Zustand、hooks、IndexedDB 操作与回调接口。

不进行全量 Tailwind 迁移，不重写状态管理，不新增账户、储蓄目标、打卡奖励或其他业务功能。

## 数据与兼容性

现有交易、预算、分类和设置数据不迁移。若历史分类存储的是 Emoji，展示层通过兼容映射尝试转换为新的图标标识；无法映射时使用通用图标。新增或编辑分类仅使用新的图标标识。

CSV 导入、重复检测、WebDAV 同步和 IndexedDB 读写逻辑保持原样。UI 重构不能改变导入结果、金额精度、日期含义或分类 ID。

## 错误与状态处理

- 表单校验在字段附近显示清晰原因，并将焦点移至首个错误字段。
- 异步保存、导入和连接测试显示进行中状态，防止重复提交。
- 成功反馈使用短时提示条或 Toast，包含明确结果。
- 导入失败、连接失败和删除确认提供文字、图标与操作，不只依赖颜色。
- 空状态、加载状态和异常状态在浅色与深色模式中均有完整样式。

## 验证计划

- 运行全部现有 Vitest 测试与 TypeScript/Vite 构建。
- 为图标名称映射、旧 Emoji 兼容映射及未知图标回退补充测试。
- 为记账校验、导航切换、月份选择和关键确认流程补充交互测试。
- 检查 390px 与 430px 宽度下的浅色、深色界面。
- 检查空数据、长商户名、长备注、大金额、超预算和导入结果状态。
- 检查键盘焦点、44px 点击区域、`aria-label` 和 `prefers-reduced-motion`。
- 手工回归记账、删除、月份切换、预算设置、CSV 导入导出与 WebDAV 配置。

## 非目标

- 不改变数据库 schema、状态管理方案或同步协议。
- 不新增账户体系、资产管理、储蓄目标或社交功能。
- 不引入网络字体、远程图片或装饰性插画依赖。
- 不进行全量 Tailwind 或组件库迁移。
- 不改变四个主页面及中央记账入口的信息架构。
